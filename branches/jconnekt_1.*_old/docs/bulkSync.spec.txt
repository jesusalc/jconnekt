Bulk Syncronization
------------------------------------------------------

This is how to synchronize users in many numbers...

Basic How To..
---------------------------
	1.can only work with one ExApp at a time
	2.get Users from ExApp
	3.update not-synced users(ExApp) to Joomla
	4.send not-synced users(Joomla) to ExApp
	5.Handle conflicts(same username in both ExApp and Joomla)
	
Detailed Way of doing..
-----------------------------
	01.get a List of Users of ExApp [ExUsers]
	02.get a list of Users already synced with ExApp [syncUsers]
	03.get a list of Joomla users [JUsers]
	04.extract list of Joomla Users new to ExApp [toEx]
	   	done by [JUsers-syncUsres]
	05.extract a list of ExApp users new to Joomla [toJ]
		done by [ExUsers-syncUsers]
	06.get the user-details of toJ from ExApp
	07.sync them to Joomla
		do not trigger other ExApp of this chages (can be done via a session variable)
		add to syncUsers table
	08.sync toEx with ExApp
		send toEx users to ExApp
		add to syncUsers table
	09.get the list of users common to toEx and toJ [conflicts]
	10.ask user to do with conflicts and store that
		options
		--------
			1.preserve Joomla (update ExApp user to be Joomla)
			2.preserve ExApp
			3.ignore
	11.do the user-given actions to conflicts
	
	
	system overview
	-------------------------------
	*.3 ExApp calls
		1.to get Username list
		2.to get Users with the given list
		3.to send Users
	*.Adding users (in Joomla) is done using the JUser
		1.create a session variable called OWNER="BULK_SYNC"
		2.alter user-plugin to no to trigger other ExApps when OWNER=="BULK_SYNC" 
	*.in STEP 08 we need reverting the changes when some fault happens 
		because adding to syncUsers table is done after the whole users get synced in the ExApp
		if some error in ExApp (when server rejection) syncUser table may get inconsistance.
